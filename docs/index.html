Controllers are used to control, manage and manipulate shapes in ways that Shapes are agnostic to. For instance in a graphical application, shapes may need to have their control points rendered so they may be transformed (moved, rotated etc). Controllers allow this behavior to be decoupled and customized.

Parameters
shape (Shape) The shape this controller instance is controlling.

shape

shape

controlPoints

Parameters

gy (Coordinate) Global/Screen Y coordinate of the hit point.
Returns
HitInfo: HitInfo containing the topmost shape at the given global coordinate along with control point info if any.

ControlPoint

Holds information about a particular ControlPoint on a shape. A shape can have several control points that allow different kinds of manipulations on it (eg moving, scaling, shearing, rotation etc).

new ControlPoint(x: Coord, y: Coord, pointType: Object, pointIndex: int, cursor: string, extraData: Object)
Parameters
x (Coord) X coordinate of the control point.
y (Coord) Y coordinate of the control point.
pointType (Object) Type of control point. Shape specific enum.
pointIndex (int) Shape specific control point index. Eg for SIZE type control points, there may be 8 (South, SouthWest, North, East etc) and the index could be used to indicate this particular one.
cursor (string) The cursor to be used for this ControlPoint when mouse hovers over it.
extraData (Object) Additional data specific to this control point and shape type.

HitInfo

When a specific "hit" occurs (eg by a TouchDown event) on a particular shape, the HitInfo captures information such as which "part" of a shape this hit occurred on.

new HitInfo(shape: Shape, hitType: Enum, hitIndex: int, cursor: string, controlPoint: ControlPoint)
Parameters
shape (Shape) The shape on which the hit occurred.
hitType (Enum) Type of hit. Hit and application specific enum.
Most often this could simply controlPoint.pointType.
hitIndex (int) Should be same as controlPoint.pointIndex if a controlPoint was provided.
cursor (string) The cursor to be used for this HitInfo. This is specifically provided if a control point was not provided.
controlPoint (ControlPoint) The control point, if any, the hit occurred on.

EventHandler

EventHandler interface. EventHandling in Sistine follows a two phase paradigm. First the event is sent to all handlers for validation. If any of the validators return false, the event is suppressed. After all validations have been approved the changes backed by the event are applied and the event is "triggered" and handled by the handlers. If any of the handlers now return false, the event handling is now suppressed.

new EventHandler()

globalTransform

The globalTransform tells how to convert a global coordinate into the coordinate system representing this shape. The inverse of this transform will map a point with respect to the shape back into the global coordinate system.

The globalTransform of a shape is simply the cumulative transforms applied inorder from the root shape all the way to this shape's transform.

draw

Our drawing needs to be smarter here. We currently would have sets of regions that need to be repainted. But since we may not be doing a recursive tree walking, we need to ensure that when a Shape "S" is being rendered, all its parent/ancestor nodes' transforms have already been applied.

So we have the hierarchy:

A -> B -> C | | |--- D | | |--- E | |--> F | |--- H | |--> G | |--- I |--- I |--- J |--- K |--- L

If these shapes can be arrived at any order, then we need following to be met: All ancestors are drawn first. When a node, say D is draw, its ancestors' transforms are applied.

For this optimization we definitely need a "topological" sorting of nodes, This can be achieved by simply tree walking the scene graph if the entire graph is to be drawn otherwise we need better indexing.
